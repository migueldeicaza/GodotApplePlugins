AVAudioSession
==============

**Inherits:** :ref:`RefCounted<class_RefCounted>` **<** :ref:`Object<class_Object>`

Controls the shared iOS/tvOS audio session from GDScript.

.. rst-class:: classref-introduction-group

Description
-----------

Use this helper to select one of Apple's predefined ``AVAudioSession`` categories so your Godot project can mix or duck audio according to platform expectations. It is most useful on mobile builds where the engine needs to share the audio device with the system music player or with voice chat as described in ``GameCenterGuide.md``. See Apple's audio session overview at `Apple's AVAudioSession reference <https://developer.apple.com/documentation/avfaudio/avaudiosession>`__.



You can either use the :ref:`current_category<class_AVAudioSession_property_current_category>` property or the :ref:`set_category()<class_AVAudioSession_method_set_category>` method for more control.

Sets the shared audio session category, mode, route sharing policy, and options.

\ ``category``: The audio session category to set (e.g., :ref:`PLAYBACK<class_AVAudioSession_constant_PLAYBACK>`, :ref:`RECORD<class_AVAudioSession_constant_RECORD>`).

\ ``mode``: The audio session mode (e.g., :ref:`DEFAULT<class_AVAudioSession_constant_DEFAULT>`, :ref:`GAME_CHAT<class_AVAudioSession_constant_GAME_CHAT>`).

\ ``policy``: The route sharing policy (e.g., :ref:`ROUTE_SHARING_DEFAULT<class_AVAudioSession_constant_ROUTE_SHARING_DEFAULT>`, :ref:`LONG_FORM_AUDIO<class_AVAudioSession_constant_LONG_FORM_AUDIO>`).

\ ``options``: A bitmask of :ref:`CategoryOptions<enum_AVAudioSession_CategoryOptions>` values (e.g., ``AVAudioSession.MIX_WITH_OTHERS | AVAudioSession.DUCK_OTHERS``).



Returns :ref:`@GlobalScope.OK<class_@GlobalScope_constant_OK>` on success, or :ref:`@GlobalScope.FAILED<class_@GlobalScope_constant_FAILED>` if the setting could not be applied (e.g. on macOS where this is not supported, or if an error occurs).

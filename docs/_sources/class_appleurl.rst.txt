AppleURL
========

**Inherits:** :ref:`RefCounted<class_RefCounted>` **<** :ref:`Object<class_Object>`

Represents an Apple Foundation.URL type

.. rst-class:: classref-introduction-group

Description
-----------

When working with Apple APIs, some of those APIs either return URLs or expect URLs, in particular

        this type is being exposed because Apple can return a URL when accessing files outside of

        your application sandbox, and you must call the :ref:`start_accessing_security_scoped_resource()<class_AppleURL_method_start_accessing_security_scoped_resource>` to

        begin accessing the resource and when you are done, you must call

        :ref:`stop_accessing_security_scoped_resource()<class_AppleURL_method_stop_accessing_security_scoped_resource>`.

:ref:`bool<class_bool>` **start_accessing_security_scoped_resource**\ (\ ) :ref:`🔗<class_AppleURL_method_start_accessing_security_scoped_resource>`

In an app that has adopted App Sandbox, makes the resource pointed to by a security-scoped URL available to the app.



                If this call succeeds, you can access the resource, and when you are done, you must call :ref:`stop_accessing_security_scoped_resource()<class_AppleURL_method_stop_accessing_security_scoped_resource>`.
